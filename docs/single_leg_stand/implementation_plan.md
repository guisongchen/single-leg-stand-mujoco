# MuJoCo 单腿站立 — 分阶段实现计划

## 项目概述

在 MuJoCo 仿真环境下，使用 G1 浮基人形机器人（23 自由度），实现从双足站立 → 单足过渡 → 单足支撑保持的完整行为链。

---

## 技术路线总览

建议采用**分层控制架构**：

- **高层**：质心（CoM）运动规划与状态机（三阶段切换逻辑）
- **中层**： whole-body 平衡控制器（QP-WBC，OSQP 求解）
- **低层**：关节力矩 → 解析逆动力学恢复

> 实现方可根据熟悉程度，在以下方案中选择：
> - **方案 A**：简化解析控制 — CoM 反馈 + 足端力分配 + 关节 PD
>   *（仅适用于双足阶段；单足阶段需要接触力优化，解析方案无法处理 CoP 约束）*
> - **方案 B（推荐）**：优化-based QP-WBC — 每次循环求解 QP（加速度 + 接触力 wrench，显式摩擦锥与 CoP 约束）
> - **方案 C**：数据驱动 — 先用 RL/MPC 离线求解，再蒸馏为在线控制器

## Phase 0: 基础环境搭建（前置阶段）

### 目标
建立可运行的 MuJoCo 仿真框架，加载 G1 模型，完成基础可视化与数据读取。

### 交付物
1. `env/g1_env.py`：MuJoCo 环境封装
   - 加载 `g1_23dof.xml`
   - 设置 `dt = 0.002 s`（500 Hz），重力 `[0,0,-9.81]`，摩擦 `mu=0.8`
   - 提供 `reset()` / `step(torque)` / `get_state()` 接口
2. `utils/kinematics.py`：基础运动学工具
   - CoM 位置/速度计算
   - 左右足端位置/速度（使用指定 body/site）
   - 接触力读取
3. `utils/visualization.py`：可选实时渲染与轨迹记录
4. `configs/g1_config.yaml`：机器人参数、控制增益、关节限制等配置

### 评估标准（Phase 0 退出条件）
| 检查项 | 通过标准 |
|--------|----------|
| 模型加载 | `mjModel` 成功加载，无 XML 解析错误 |
| 初始姿态合理 | 设置初始关节角后，足端无悬空/严重穿透，仿真不产生 NaN/爆炸 |
| 数据读取 | CoM 位置、足端位置、接触力读取值合理（CoM 高度 ~0.7-1.0m） |
| 帧率 | 纯仿真步进（无渲染）达到实时或更快 |
| **关节阻尼** | 所有驱动关节已配置 `damping > 0` 与 `armature > 0` |

> **说明**：Phase 0 不要求机器人"站稳"，仅要求环境基础设施可用。稳定站立属于 Phase 1 的评估范围。
>
> **重要**：G1 原始模型所有驱动关节的 `damping="0"`、`armature="0"`。零阻尼会导致速度振荡永不衰减，
> 必须在 XML 中为每个驱动关节添加 `damping="0.5" armature="0.01"`，否则 Phase 1/2 的控制器无法稳定。

## Phase 1: 双足站立稳定控制器

### 目标
实现一个能长期稳定维持双足站立的控制器，为后续过渡阶段提供可靠的初始状态。

### 技术要点
1. **参考姿态设计**
   - 轻微屈膝（膝关节 ~5-15°），降低 CoM 以提高稳定裕度
   - 髋关节 `hip_pitch ≈ -0.1 rad`：将躯干轻微后仰，抵消上半身质量导致的 CoM 前移
   - 髋关节、踝关节适当补偿，使躯干保持直立
   - 双足与肩同宽或略宽
   - **必须先确认** XML 中所有驱动关节已配置 `damping` 与 `armature`（见 Phase 0）

2. **CoM 稳定控制**
   - 期望 CoM 水平位置固定在双足中点上方
   - 采用线性反馈：
     ```
     a_com_des = Kp * (p_com_target - p_com) + Kd * (0 - v_com)
     ```
   - 将期望 CoM 加速度映射为足端力分配（双足支撑时按压力分配）

3. **力分配与逆动力学**
   - 双足接触时，总接触力需支撑重力 + 产生期望 CoM 加速度
   - 法向力分配建议：左右脚各 ~50%（或根据 CoM 偏移微调）
   - 切向力由 CoM 水平加速度决定，需满足摩擦锥约束
   - 可选：用小型 QP 求解最优力分配 + 零净力矩点（ZMP）约束

4. **姿态控制**
   - 躯干方向 PD 控制（保持直立）
   - 摆动脚（未来阶段）当前保持与地面接触力

5. **关节级执行**
   - 由期望足端力 + 姿态任务，通过 QP 或解析方式求解关节加速度/力矩
   - 或简化：在关节空间做 PD 跟踪，配合重力补偿

### 交付物
- `controllers/bipedal_stance_controller.py`：双足站立控制器
- `scripts/test_bipedal_stance.py`：独立测试脚本，可运行 N 秒并记录指标
- 数据日志：`logs/bipedal_*.npz`（CoM 轨迹、足端位置、接触力、力矩）

### 评估标准（Phase 1 退出条件）
| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 稳定站立时间 | >= 10 s | 单次仿真运行 10s，目测/自动判据不摔倒 |
| 足端滑移 | < 0.005 m | 记录左右足端位置 RMSE 相对于初始位置 |
| CoM 位置 RMSE | < 0.02 m | CoM 水平位置与目标位置（双足中点）的 RMSE |
| 基座姿态发散 | < 5°（俯仰/横滚） | 躯干相对世界坐标系的倾角 |
| 接触力连续性 | 双脚法向力始终 > 10 N | 无瞬时离地 |
| 控制实时性 | 求解时间 < 0.002 s | 每步统计耗时，取 99th percentile |

## Phase 2: 双足 → 单足过渡控制器

### 目标
在双足稳定的基础上，设计状态切换与过渡轨迹，使机器人将重心转移到支撑脚并平稳抬起摆动脚。

### 技术要点
1. **状态机设计**
   ```
   BIPEDAL_STANCE → WEIGHT_SHIFT → LIFT_OFF → SINGLE_STANCE
   ```
   - `WEIGHT_SHIFT`：CoM 水平移动至支撑脚上方（1-2 秒）
   - `LIFT_OFF`：摆动脚逐渐减重并抬起（0.5-1 秒）
   - 切换判据：基于 CoM 位置误差、摆动脚接触力阈值、计时器

2. **CoM 轨迹规划**
   - 从双足中点平滑移动到支撑脚正上方
   - 使用三次/五次样条或线性插值 + 平滑滤波
   - 垂直方向保持恒定或轻微调整以保持腿长

3. **足端力分配（Weight Shift 阶段）**
   - 随时间连续改变法向力分配比：50/50 → 90/10 → 100/0
   - 力分配比由 CoM 目标轨迹隐式决定或显式插值

4. **摆动脚轨迹**
   - 抬起高度：5-10 cm（避免与地面摩擦即可）
   - 轨迹：垂直抬起 → 微收向支撑腿（可选）→ 保持悬停
   - 使用最小加加速度（jerk-minimal）轨迹

5. **过渡稳定性保障**
   - 过渡期间 ZMP/CoM 必须始终保持在支撑多边形内
   - 支撑脚法向力始终 > 10 N
   - 切换速度不宜过快，避免惯性冲击

### 交付物
- `controllers/transition_controller.py`：过渡阶段控制器
- `controllers/state_machine.py`：三阶段状态机管理
- `planners/com_planner.py`：CoM 轨迹规划器
- `planners/swing_foot_planner.py`：摆动脚轨迹规划器
- `scripts/test_transition.py`：独立测试脚本

### 评估标准（Phase 2 退出条件）
| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 过渡过程不摔倒 | 全程基座高度 > 0.5m，俯仰/横滚 < 15° | 自动判据 |
| 摆动脚离地平滑性 | 离地后摆动脚高度持续 > 0.02m | 高度轨迹检查 |
| 支撑脚稳定性 | 支撑脚滑移 < 0.005 m | 位置 RMSE |
| CoM 到位精度 | 过渡结束后 CoM 距支撑脚中心 < 0.05 m | 最终位置误差 |
| 过渡时间 | 建议 1.5 - 3.0 s（可调节） | 记录日志 |
| 控制实时性 | 求解时间 < 0.002 s | 99th percentile |

## Phase 3: 单足支撑保持控制器

### 目标
在单足支撑状态下长期维持平衡，满足至少 **15 秒**的稳定保持要求。

> **说明**：3 秒太短，无法暴露慢漂移失稳（如角动量累积）。实际调试中发现机器人在 ~18 s 才因角动量发散而倾倒，
> 因此工程验收标准必须覆盖长时域行为。

### 技术要点
1. **支撑脚选择**
   - 默认右脚或左脚，可在配置中切换
   - 过渡阶段结束时确定支撑脚，后续不再切换

2. **单足 CoM 控制**
   - 目标 CoM 固定在支撑脚接触矩形中心上方（非踝关机原点），以最大化 CoP 裕度
   - 反馈增益通常比双足阶段更积极（单足支撑裕度更小）
   - 可采用 LIPM（线性倒立摆）模型设计反馈律

3. **角动量/姿态控制**
   - **单足阶段必须引入质心角动量（CAM）调节任务**。仅做躯干姿态 PD 是不够的；
     弱 CAM 增益会导致角动量缓慢累积，CoM 在 10–20 s 后漂出支撑多边形。
   - CAM 任务与躯干姿态任务并存：前者管整体旋转动量，后者管局部朝向。
   - **不要单纯增大躯干姿态增益来“压”稳定**——过高的姿态加速度指令会超出 CoP 包络，
     反而加速倾倒（实测 `pelvis_kp` 从 30 提到 80 后，倾倒时间从 18 s 缩短到 10 s）。

4. **摆动脚管理**
   - 保持离地（> 0.02m）
   - **仅做 z 向软跟踪**（`J_swing[2:3]`）。xy 向跟踪会把摆动腿与躯干平衡耦合，
     摆动腿的惯性反力通过骨盆传递到支撑脚，引发慢振荡。
   - 可自然悬垂或轻微回收（减小转动惯量）

5. **支撑脚速度阻尼**
   - 在 QP 等式约束中为支撑脚加入 `accel_offset = -kd * support_vel`
   - 防止脚在地面上微滑导致的位姿漂移

### 交付物
- `controllers/transition_controller.py`（已包含单足保持逻辑，无需单独文件）
- `scripts/debug_single_leg.py`：单足阶段诊断脚本（记录 CoP、CAM、力利用率）

### 评估标准（Phase 3 退出条件）
| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 单足稳定时间 | >= **15.0 s** | 从进入 SINGLE_LEG 开始计时 |
| 支撑脚滑移 | < 0.005 m | 足端位置 RMSE |
| 支撑脚接触力 | 法向力持续 > 10 N | 力日志检查 |
| 摆动脚离地 | 高度持续 > 0.02 m | 高度日志检查 |
| 基座姿态发散 | 俯仰/横滚 < 10° | 姿态角日志 |
| CoM 位置误差 | 与目标 CoM 水平误差 < 0.05 m | RMSE |
| 控制实时性 | 求解时间 < 0.002 s | 99th percentile |

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| G1 模型初始姿态不稳定 | Phase 0/1 受阻 | 调整初始关节角（屈膝 `knee=0.15 rad`、髋后仰 `hip_pitch=-0.1 rad`）；**必须在 XML 中添加关节 damping + armature** |
| CoM 反馈增益难调 | Phase 1 振荡/软塌 | 先做单刚体仿真测试，再迁移到全身；分 Kp/Kd 独立调参 |
| 过渡阶段频繁摔倒 | Phase 2 受阻 | 大幅降低过渡速度；增加过渡子阶段数量；强制过渡期间 CoM 在支撑面内；摆动脚卸载时添加向上 `accel_offset` 避免 QP 不可行 |
| 单足慢漂移（10–20 s 后倾倒） | Phase 3 看似通过 3s 实则失败 | **优先检查 CAM（质心角动量）是否被主动调节**，而非单纯增大躯干姿态增益。实测 `pelvis_kp` 从 30 提到 80 后倾倒时间从 18 s 缩短到 10 s |
| QP 求解超时或不可行 | 实时性/稳定性不达标 | OSQP 已足够快（~0.3 ms/步）；若不可行，检查是否要求固定已无接触的脚，或摩擦锥/CoP 约束过紧 |
| 足端滑移过大 | 滑移指标失败 | 增大摩擦系数（先调 mu=1.0 测试）；检查法向力是否充足；在支撑脚约束中加入速度阻尼 `accel_offset = -kd * vel` |
